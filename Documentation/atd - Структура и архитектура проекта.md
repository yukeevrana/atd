# Структура и архитектура проекта

Проект задуман несложным первым проектом на Unity, однако, было принято решение сразу начать применять практики, которые хотелось бы применить в будущем на более сложных проектах. В основном, это значит следующее:

- Как можно большее количество логики игры должно быть написано отдельно от движка Unity
- ECS
	- Игровые объекты (сущности) связаны с наборами данных (компонентами)
	- Системы обрабатывают компоненты
	- Обработка пакетная, это значит, что система обрабатывает все компоненты за раз (асинхронно или нет)
	- Системы, по возможности, работают с чистыми данными, без упоминания частей движка Unity
